PREGUNTAS FRECUENTES

General

¿Para qué sirven los servicios web?

Para intercambiar datos entre aplicaciones o sistemas. Las aplicaciones escritas en varios lenguajes de programación que funcionan en plataformas diferentes pueden utilizar web services para intercambiar información a través de una red.

¿Cómo funciona un servidor web?

El provedor de servicio genera el WSDL y lo registra en el directorio UDDI. El solicitante del servicio o la aplicación quiere un servicio web y se pone en contacto con el UDDI para localizar el web service. El cliente, basándose en la descripción descrita por el WSDL, envía una solicitud para un servicio particular al Web Service Listener, que se encarga de recibir y enviar los mensajes en formato SOAP. El Web Service analiza el mensaje SOAP de la solicitud e invoca una operación particular en la aplicación para procesar la solictud. El resultado se escribe de nuevo en SOAP en forma de respuesta y se envía al cliente. El cliente analiza el mensaje de respuesta SOAP y lo interpreta o genera un error si ha habido alguno.

¿Cómo compruebo que el servicio web está funcionando correctamente?

En Postman copiamos la URL del service web, depende del método usado le mandamos los parámetros necesarios. Si el servicio está funcionando correctamente nos devolvera el código de estado 200:OK, de no ser el caso, nos devolverá códigos de error que pueden ser: 404:NO FOUND.

Features

¿Qué programas necesito para implementar un servicio web?

Para desarrollar los archivos php de los web service se utilizará un editor de código que puede ser Visual Code o Sublime text, para comprobar la funcionalidad de los web service se usará el software Postman y para gestionar la base de datos necesitaremos XAMPP

¿Qué debo hacer si me sale el error 404:NO FOUND?

En la mayoría de casos en lo que ocurre este error es porque los nombre de las tablas o de los campos de la tabla están mal escritos y no coinciden con la base de datos, lo que se debe hacer es verificar si están escritos correctamente.

¿Qué métodos HTTP se utilizará en los web service?

En la implementación de los web service se utilizará los siguientes métodos:
HTTP Verb CRUD Entire Collection (e.g. /usuario) Specific Item (e.g. /usuario/{id})
POST Create 201 (Created), 'Location' header con enlace /usuario/{id} contiene una nueva identificación. 404 (Not Found)
GET Read 200 (OK), lista de datos. Utilice algún paráme y filtro para navegar por listas grandes. 200 (OK), un solo dato. 404 (Not Found), si el ID no se encuentra o es inválido.
PUT Update/Replace 405 (Method Not Allowed), cuando desee reemplazar los datos 200 (OK) or 204 (No Content). 404 (Not Found), si el ID no se encuentra o es inválido.
PATCH Update/Modify 405 (Method Not Allowed), cuando desee reemplazar la colección en sí. 200 (OK) or 204 (No Content). 404 (Not Found), si el ID no se encuentra o es inválido.
DELETE Delete 405 (Method Not Allowed), si desea eliminar todos los datos. 200 (OK). 404 (Not Found), si el ID no se encuentra o es inválido.
